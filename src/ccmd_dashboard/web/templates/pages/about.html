AOR tagging

Articles are mapped to combatant commands by combining spaCy NER output (GPE/ORG/PERSON/LOC/FAC/EVENT/NORP entities) with keyword matching against config/ccmd_aor.yaml. Entity matches are weighted 2× because the NER model has already resolved ambiguity; keyword matches are cheaper but noisier. An article may map to 0, 1, or many CCMDs. Everything below the score threshold goes to the Unassigned bucket.

MDM scoring

Two-stage, explainable. Stage 1 (LLM) returns structured features — verifiable claims, emotional language, fallacies, unsourced assertions, temporal claims, source transparency. Stage 2 (deterministic, code in classify/scoring.py) combines those features with source tier, state-affiliation, and corroboration count to produce a concern score. Every sub-signal and its weight is stored in the assessment's reasoning breakdown and surfaced in the UI.

Not in scope for this prototype

  • No paste-in analysis of arbitrary text.
  • No real-time alerting.
  • No .mil data, no CUI, no PII.
